# Shield Fencing SEO Audit & Implementation Instructions

**Audit Date:** 9 May 2026
**Website:** https://shieldfencing.com.au
**Stack:** Next.js 15 / React 19 / Tailwind / Vercel

---

## EXECUTIVE SUMMARY

Shield Fencing's website is well-built with strong foundational SEO: unique title tags and meta descriptions on every page, canonical URLs, proper Open Graph tags, a solid blog content library (30+ posts), and FAQ sections with keyword-rich answers on service pages. The site is fast (Next.js SSR on Vercel), mobile-responsive, and uses semantic HTML.

However, there are significant gaps holding the site back from ranking higher in Brisbane-area fencing searches. The three most impactful issues are: (1) no sitemap.xml or robots.txt file, meaning Google is guessing your site structure; (2) zero structured data / schema markup, meaning you're missing rich snippets for FAQ, LocalBusiness, and reviews; and (3) the homepage is thin on crawlable text content, with the strongest SEO copy buried inside the service pages rather than linked back to the homepage.

Fixing these three issues alone would materially improve your visibility in Google Search for Brisbane fencing queries.

## CRITICAL ISSUES (High Priority)

### 1. No sitemap.xml

**Problem:** There is no sitemap.xml file — not in `/public/`, not generated by Next.js config, and no `sitemap.ts` route exists.

**Why it matters:** Google relies on sitemaps to discover pages efficiently, especially for newer sites. Without one, blog posts and service pages may be slow to index or missed entirely.

**Fix — add `src/app/sitemap.ts`:**

```ts
import { MetadataRoute } from 'next'
import { getAllPosts } from '@/lib/posts'

export default function sitemap(): MetadataRoute.Sitemap {
  const baseUrl = 'https://shieldfencing.com.au'
  const posts = getAllPosts()

  const staticPages = [
    { url: baseUrl, lastModified: new Date(), changeFrequency: 'weekly' as const, priority: 1.0 },
    { url: `${baseUrl}/colorbond`, lastModified: new Date(), changeFrequency: 'monthly' as const, priority: 0.9 },
    { url: `${baseUrl}/timber`, lastModified: new Date(), changeFrequency: 'monthly' as const, priority: 0.9 },
    { url: `${baseUrl}/retaining-walls`, lastModified: new Date(), changeFrequency: 'monthly' as const, priority: 0.9 },
    { url: `${baseUrl}/reviews`, lastModified: new Date(), changeFrequency: 'weekly' as const, priority: 0.8 },
    { url: `${baseUrl}/about`, lastModified: new Date(), changeFrequency: 'monthly' as const, priority: 0.7 },
    { url: `${baseUrl}/blog`, lastModified: new Date(), changeFrequency: 'weekly' as const, priority: 0.7 },
    { url: `${baseUrl}/contact`, lastModified: new Date(), changeFrequency: 'yearly' as const, priority: 0.5 },
    { url: `${baseUrl}/privacy`, lastModified: new Date(), changeFrequency: 'yearly' as const, priority: 0.2 },
  ]

  const blogPages = posts.map((post) => ({
    url: `${baseUrl}/blog/${post.slug}`,
    lastModified: new Date(post.date),
    changeFrequency: 'yearly' as const,
    priority: 0.6,
  }))

  return [...staticPages, ...blogPages]
}
```

### 2. No robots.txt

**Problem:** No `robots.txt` file exists. Google may crawl pages inefficiently.

**Fix — add `src/app/robots.ts`:**

```ts
import { MetadataRoute } from 'next'

export default function robots(): MetadataRoute.Robots {
  return {
    rules: {
      userAgent: '*',
      allow: '/',
      disallow: ['/api/', '/thank-you'],
    },
    sitemap: 'https://shieldfencing.com.au/sitemap.xml',
  }
}
```

**Fix — add FAQPage schema to each service page.** Create a reusable component `src/components/FaqSchema.tsx`:

```tsx
interface FaqSchemaProps {
  faqs: { q: string; a: string }[]
}

export default function FaqSchema({ faqs }: FaqSchemaProps) {
  const schema = {
    '@context': 'https://schema.org',
    '@type': 'FAQPage',
    mainEntity: faqs.map((faq) => ({
      '@type': 'Question',
      name: faq.q,
      acceptedAnswer: {
        '@type': 'Answer',
        text: faq.a,
      },
    })),
  }

  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(schema) }}
    />
  )
}
```

Then import and use `<FaqSchema faqs={faqs} />` in `colorbond/page.tsx`, `timber/page.tsx`, `retaining-walls/page.tsx`, and `ServicePageLayout.tsx`.

## TECHNICAL SEO RECOMMENDATIONS

### 4. Font Mismatch — Brand vs Implementation

**Problem:** Brand guidelines specify Montserrat (primary), Open Sans (secondary), Merriweather (tertiary). The website loads **Inter** instead.

**Impact:** Not an SEO issue per se, but brand consistency matters. More importantly, loading unused fonts wastes bandwidth.

**Fix:** Either update `layout.tsx` to load Montserrat + Open Sans, or update the brand guidelines. If keeping Inter, remove the brand guideline reference to avoid confusion.

### 5. Missing Open Graph Image

**Problem:** No `og:image` is set in the global metadata or on any page. When the site is shared on Facebook, LinkedIn, or iMessage, there's no preview image.

**Fix in `layout.tsx` metadata:**

```ts
openGraph: {
  // ...existing fields
  images: [
    {
      url: 'https://shieldfencing.com.au/og-image.jpg',
      width: 1200,
      height: 630,
      alt: 'Shield Fencing - Brisbane Fencing & Retaining Wall Specialists',
    },
  ],
},
```

Create a 1200x630px branded OG image and place it at `public/og-image.jpg`.

### 6. Contact Page Set to noindex

**Problem:** `/contact` has `robots: { index: false }`. Contact pages are valuable for local SEO — they contain your phone number, service area, and credentials.

**Fix:** Remove `robots: { index: false }` from `src/app/contact/page.tsx` metadata. The contact page should be indexable.

### 7. Get-a-Quote Page noindex is Correct

The `/get-a-quote` page has `robots: { index: false }` which is correct — form-only pages shouldn't be indexed.

### 8. No 404 Page

**Problem:** No custom `not-found.tsx` exists. A custom 404 page with links to services and the quote form helps retain users who land on broken links.

**Fix:** Create `src/app/not-found.tsx` with links back to services and the enquiry form.

### 9. No Image Alt Text on Service Profile Images

**Problem:** The Colorbond profile images use basic alt text like `"Sawtooth Colorbond profile"` — this is acceptable but could be more descriptive for image search.

**Fix:** Use more descriptive alt text:
- `"Sawtooth Colorbond steel fencing profile installed in Brisbane"`
- `"Trimline Colorbond fence panel close-up showing smooth finish"`
- `"Ezyline flat Colorbond fencing profile for modern Brisbane homes"`

### 10. No `next/image` Width/Height on Profile Images

**Problem:** Profile images use `fill` mode which is fine but there are no explicit `sizes` props. This means the browser may download larger images than needed on mobile.

**Fix:** Add `sizes` prop:
```tsx
<Image
  src={p.image}
  alt={...}
  fill
  sizes="(max-width: 768px) 100vw, 33vw"
  className="object-contain scale-150"
/>
```

## CONTENT STRATEGY RECOMMENDATIONS

### Missing Service Area / Suburb Pages

**Opportunity:** The site has a massive suburb list in `qld-suburbs.ts` but it's only used for the interactive map. Creating suburb-specific landing pages (or at minimum, a dedicated service areas page) would capture long-tail local searches like "fencing Carindale", "retaining wall Kenmore", "Colorbond fence Springfield".

**Recommendation:**
1. Create a `/service-areas` index page listing all regions with suburb names
2. Consider creating regional cluster pages: `/service-areas/north-brisbane`, `/service-areas/logan`, `/service-areas/ipswich` etc.
3. Each cluster page should mention 10-15 suburbs in that region naturally

### Blog Content Gaps

The current 30+ posts are well-targeted. Missing topics to fill gaps:

1. **"Do I need a permit for a fence in Brisbane?"** — High search volume, council-related query
2. **"Best fence for windy areas in Brisbane"** — Targets storm-prone suburbs
3. **"Fence on a retaining wall: What you need to know"** — You do this work, no dedicated guide
4. **"How to choose between concrete and timber sleepers"** — Decision-stage content
5. **"Colorbond fence gate options and costs Brisbane"** — Gates are a common add-on
6. **"What is a good neighbour fence? Brisbane guide"** — Specific style query
7. **"Lapped and capped fence cost Brisbane"** — Price-intent long-tail keyword

### Internal Linking Improvements

**Problem:** Service pages link to blog posts at the bottom, but blog posts don't consistently link back to service pages or to each other.

**Fix:** In each blog post, ensure at least 2 internal links:
- One to the relevant service page (e.g., blog about Colorbond costs links to `/colorbond`)
- One to the quote form (`/get-a-quote`)
- Ideally one to a related blog post
